## Description
The **ft_printf** project is a fundamental assignment in the 42 curriculum that challenges students to recode the standard C library `printf()` function. The primary goal is to understand and master the usage of variadic functions in C using the `<stdarg.h>` macros (`va_start`, `va_arg`, `va_copy`, and `va_end`). 

This implementation mimics the basic behavior of the original `printf()`, handling the mandatory format specifiers (`%c`, `%s`, `%p`, `%d`, `%i`, `%u`, `%x`, `%X`, and `%%`) without implementing the original's buffer management.

## Instructions

### Compilation
The project compiles into a library named `libftprintf.a` using the `ar` command. The provided Makefile includes the following standard rules:
* `all`: Compiles the source files and generates the library.
* `clean`: Removes the object files (`.o`).
* `fclean`: Removes the object files and the compiled library (`libftprintf.a`).
* `re`: Cleans and recompiles everything from scratch.

To compile the library, run the following command in your terminal:
```bash
make
```
Usage
To use ft_printf in your own C projects, include the header file in your code and link the compiled library during compilation. Creat a 'main.c' file:

```bash
#include "ft_printf.h"

int main(void)
{
    ft_printf("Hello, %s! Number: %d\n", "World", 42);
    return (0);
}
```

### In the terminal type:

```bash
cc main.c -L. -lftprintf -o my_program
./my_program
```

### Algorithm and Data Structure Justification
Chosen Structure:

The project relies on a single-pass stream parsing approach combined with a structural driver function `(ft_conversion)` acting as a router.

### Justification
* Why this approach? Since ft_printf processes characters sequentially from a format string, an iterative character-by-character scan is the most efficient method ($O(N)$ time complexity).

* The Router Pattern: When a % character is detected, the control flow delegates the argument extraction to ft_conversion. This approach treats each conversion type independently, fulfilling the subject's requirement for a well-structured and highly extensible architecture.

* Variadic Variable Tracking: A va_list structure is passed by reference/value down to the router. This avoids complex state tracking data structures (like custom structs or arrays), keeping the memory footprint at $O(1)$ auxiliary space and eliminating risks of unexpected heap allocation memory leaks.
